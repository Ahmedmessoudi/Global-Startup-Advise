# STARTUP COUNTRY ADVISOR — FULL PROJECT SPECIFICATION
## For AI Agents & Developers

---

## 1. PROJECT OVERVIEW

A Streamlit-based data mining application that helps entrepreneurs evaluate startup opportunities across countries using the CIA World Factbook dataset. The user interacts via a chat interface, describes their startup idea and a target country, and the system runs a full data mining pipeline to return:

- Opportunity scores and rankings
- Interactive charts and visualizations
- Text insights generated by an LLM (Groq API)
- Alternative country suggestions based on clustering

## 6. DATA MINING PIPELINE (data_pipeline.py)

### Step 1 — Data Loading
- Load `countries.csv` with pandas
- Select relevant columns (see Section 3)
- Log shape, dtypes

### Step 2 — Data Cleaning
- Strip currency symbols: `$`, `,`, `%`, text annotations like `(2020 est.)`
- Convert all numeric columns to float using regex extraction
- Identify and handle missing values:
  - If column missing > 60%: drop column
  - Otherwise: impute with median
- Remove rows where country is null
- Standardize country names (strip whitespace, title case)

### Step 3 — Feature Engineering
Build composite indicators from raw columns:

| Feature | Formula |
|---------|---------|
| `market_size_score` | log(Population) × log(GDP_PPP) |
| `digital_score` | Internet_users / Population × 100 |
| `economic_stability` | 100 - Inflation_rate - (Public_debt/10) |
| `growth_potential` | GDP_growth_rate × (100 - Unemployment_rate) / 100 |
| `infrastructure_score` | (Electricity_production + Mobile_phones) / Population |
| `trade_openness` | (Exports + Imports) / GDP_PPP × 100 |
| `human_capital` | Literacy × (1 + GDP_per_capita/100000) |

### Step 4 — Normalization
- Apply MinMaxScaler to all feature columns → values in [0, 1]
- Store scaler for inverse transform if needed

### Step 5 — Country Clustering (KMeans)
- Input: all 7 normalized features
- k=5 clusters (configurable)
- Assign cluster label to each country
- Name clusters heuristically:
  - Highest avg GDP per capita → "Advanced Economies"
  - Highest growth → "Emerging Markets"
  - Lowest scores overall → "Frontier Markets"
  - High digital + low GDP → "Digital Leapfrog"
  - Balanced mid-tier → "Developing Economies"

### Step 6 — Opportunity Scoring
- Compute weighted sum of features based on startup sector
- See Section 7 for sector weights
- `opportunity_score` = Σ(weight_i × feature_i) for i in features
- Rank all countries by opportunity_score (desc)

## 8. GROQ AGENT (groq_agent.py)

### 8A — Intent Parsing
Send user message to Groq with system prompt:
```
You are a startup advisor assistant. Extract from the user's message:
1. startup_description: brief description of the startup
2. target_country: the country they want to launch in (normalize to standard name)
3. sector: one of [fintech, e-commerce, edtech, healthtech, agritech, saas, logistics, general]

Respond ONLY in JSON format:
{"startup_description": "...", "target_country": "...", "sector": "..."}
```
*Note: The agent applies robust regex-based extraction to parse the JSON object, allowing it to work reliably with reasoning LLMs (like `qwen-qwq-32b`) that output `<think>` blocks before the JSON.*

### 8B — Results Synthesis
After running the data mining pipeline, send Groq a structured context with:
- Target country metrics (all 7 features, raw values, rank)
- Top 5 alternative countries with scores
- Cluster membership and cluster description
- Startup description and sector

System prompt for synthesis:
```
You are an expert startup advisor and data analyst. You have access to CIA World Factbook 
data for {N} countries. A user wants to launch a {sector} startup ({description}) in {country}.

Based ONLY on the data provided below, write a structured analysis:
1. OPPORTUNITY ASSESSMENT for {country} (2-3 sentences, mention specific metrics)
2. KEY RISKS (2-3 bullet points based on weak metrics)  
3. KEY OPPORTUNITIES (2-3 bullet points based on strong metrics)
4. ALTERNATIVE COUNTRIES (explain why each of the top alternatives is interesting)
5. RECOMMENDATION (clear verdict: Recommended / Proceed with Caution / Not Recommended)

DATA:
{json_data}

Be specific, cite numbers from the data. Do not invent information not in the data.
```

### 8C — Follow-up Chat
Maintain conversation history. Each follow-up message includes:
- Full system prompt with dataset context
- Previous messages
- New user message

The LLM can answer follow-up questions, compare countries, explain metrics — all grounded in the dataset.

## 9. RESULTS ENGINE (results_engine.py)

### Functions:
- `fuzzy_find_country(name, df, threshold)` → robustly matches country names despite typos or variations
- `get_country_profile(country_name, df)` → dict of all metrics for one country
- `get_top_countries(df, sector, n=10)` → top N countries by opportunity score for sector
- `get_similar_countries(country_name, df, n=5)` → countries in same cluster, sorted by score
- `get_country_rank(country_name, df, sector)` → global rank of target country
- `get_feature_percentiles(country_name, df)` → percentile rank per feature (for radar chart)
- `compare_countries(country_list, df)` → side-by-side comparison dict
- `build_llm_context(target_country, sector, ...)` → builds structured context required for Groq synthesis

### State Management (st.session_state):
- `messages`: list of {role, content} dicts
- `analysis_results`: dict from last pipeline run
- `df_clean`: cached cleaned dataframe
- `groq_api_key`: user-provided key
- `last_country`: last analyzed country
- `last_sector`: last analyzed sector

### App Flow:
1. User enters Groq API key in sidebar
2. Dataset loads and cleans on first run (cached with @st.cache_data)
3. User types in chat box
4. On submit:
   a. Parse intent (Groq)
   b. Run pipeline
   c. Generate charts
   d. Synthesize response (Groq)
   e. Display in chat + update Analysis/Map tabs
5. Follow-up questions → Groq answers using cached results context

## 15. DATA MINING GLOSSARY (for display in UI)

- **Data Cleaning:** Removing/imputing missing values, standardizing formats
- **Feature Engineering:** Creating composite scores from raw indicators
- **Normalization:** Scaling all features to [0,1] for fair comparison
- **K-Means Clustering:** Grouping countries with similar economic profiles
- **Opportunity Scoring:** Weighted sum of features tuned to startup sector
- **PCA (Principal Component Analysis):** Reducing 7D feature space to 2D for visualization
- **Percentile Rank:** Country's position relative to all countries (0=worst, 100=best)
